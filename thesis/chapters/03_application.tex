\section{Metal-Organic Framework Synthesis Using Data Models}\label{sec:application:metal}

Metal-Organic Framework (MOF) synthesis refers to the stepwise preparation of crystalline coordination materials from metal precursors and organic linkers under controlled laboratory conditions. As emphasized by Neubauer et al. \cite{neubauer2026data}, the synthesis process is not only defined by the final product, but also by the full protocol context (reagents, roles, action sequence, and process conditions), which must be represented in a structured and machine-readable way for analysis and reuse.

Listings~\ref{lst:application_mof_json_example_s_1} and~\ref{lst:application_mof_json_example_s_2} represent a structured subset of the MOF synthesis dataset discussed. It models synthesis experiments as process-centric records under \texttt{Synthesis[*]}, where each record combines (i) experimental setup (\texttt{Hardware}), (ii) run-level metadata including product phase (\texttt{Metadata}), (iii) ordered procedural actions across \texttt{Prep}, \texttt{Reaction}, and \texttt{Workup}, and (iv) a reagent inventory with semantic roles such as substrate, ligand, and solvent.

In these excerpts, each synthesis run (e.g., \texttt{S-1}, \texttt{S-2}) is represented as a complete protocol instance. The data captures not only which chemicals are used, but also how they are used: quantities, processing actions (e.g., \texttt{Add}, \texttt{Sonicate}, \texttt{HeatChill}, \texttt{Dry}), and operating conditions such as time, temperature, and pressure. Therefore, the JSON represents executable laboratory knowledge in machine-readable form, rather than only final product labels.

\begin{lstlisting}[float=htbp,style=json,caption={Part of RML mapping to convert data in Listing~\ref{lst:application_mof_json_example_s_1} and Listing~\ref{lst:application_mof_json_example_s_2}},label={lst:application_mof_rml_example_1}, focusing on Hardware and Metadata data]
{
@prefix rr: <http://www.w3.org/ns/r2rml#> .
@prefix rml: <http://semweb.mmlab.be/ns/rml#> .
@prefix ql: <http://semweb.mmlab.be/ns/ql#> .
@prefix ex: <http://example.org/> .
@prefix chmo: <http://purl.obolibrary.org/obo/CHMO_> .
@prefix obo: <http://purl.obolibrary.org/obo/> .
@prefix rdfs: <http://www.w3.org/2000/01/rdf-schema#> .

<#SynthesisSource> rml:source "Data.json" ; rml:referenceFormulation ql:JSONPath ; rml:iterator "$.Synthesis[*]" .

<#SynthesisTriplesMap>
  rml:logicalSource <#SynthesisSource> ;
  rr:subjectMap [ rr:template "http://example.org/synthesis/{Hardware.Component[0]._id}" ; rr:class chmo:0000410 ] ;
  rr:predicateObjectMap [ rr:predicate ex:componentID ; rr:objectMap [ rml:reference "Hardware.Component[0]._id" ] ] ;
  rr:predicateObjectMap [ rr:predicate ex:componentType ; rr:objectMap [ rml:reference "Hardware.Component[0]._type" ] ] ;
  rr:predicateObjectMap [ rr:predicate rdfs:label ; rr:objectMap [ rml:reference "Metadata._description" ] ] ;
  rr:predicateObjectMap [ rr:predicate chmo:hasProduct ; rr:objectMap [ rml:reference "Metadata._product" ] ] .
}
\end{lstlisting}
